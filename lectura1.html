Control de Versiones

Imagen tomada de Herramientas de control de versiones

Es importante antes de comenzar con el diseño de un sitio y el uso de herramientas como Github entender el concepto de control de versiones

¿Que es un sistema de control de versiones?

Es un medio usado para controlar los cambios que se realizan en el código de diferentes aplicaciones, esto se usa con el fin de que las aplicaciones que son desarrolladas por un grupo de desarrolladores no tengan diferentes versiones de código usadas por cada uno de los colaboradores, con este tipo de sistemas se puede tener un verdadero trabajo colaborativo que permita que todos puedan tener un unico repositorio de archivos controlado y tambien un sistema en el cual, en caso de tener algun error con la aplicacion por el cambio en el código se pueda volver a versiones anteriores.

¿Que tipos de sistemas hay?

Existen sistemas de control de versiones locales, centralizados y distribuidos. Lo más recomendable hoy en día es usar sistemas centralizados que no dependan de un solo equipo para poder usarlos. Además, es claro que con un sistema de control de versiones centralizado, varios desarrolladores pueden trabajar al mismo tiempo.

Los mejores sistemas de control de versiones son los distribuidos, los cuales permiten tener una copia del repositorio en el computador del desarrollador. Esto hace que las aplicaciones que tienen una gran cantidad de desarrolladores cada uno de ellos pueda hacer su desarrollo de una manera particular y luego subir sus avances al repositorio en la nube.

Otra de las ventajas de tener el repositorio local mientras se esté trabajando es que el desarrollador no depende de que el repositorio este en funcionamiento para que pueda trabajar en su aplicación. Solamente depende del repositorio cuando va a subir sus nuevos adelantos.

Software de control de versiones

Ahora les vamos a mostrar cuales son las aplicaciones de control de versiones más usadas:

CVS: El cual permite concurrencia de usuarios, pero esta limitado ya que solo permite archivos con contenido diferente a ASCII

Git: Tiene bastantes ventajas, como por ejemplo, que el es posible compartir con otros desarrolladores cierta parte del código que tipo de permisos se le pueden dar (consultar, agregar notas, modificar el código). Una desventaja es que no es tan intuitivo para su uso.

Subversion: Ocupa mucho mas espacio en disco para guardar el repositorio, pero su interfaz de usuario es mucho mas amigable que las que usa Git
