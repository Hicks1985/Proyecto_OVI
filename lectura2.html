GITHUB - HERRAMIENTA DE CONTROL DE VERSIONES

Imagen tomada de Curso de GitHub

Github es la herramienta más usada para el manejo de control de versiones de repositorios de tipo Git. Aunque no es tan intuitiva como esperamos, una vez se conoce su funcionamiento, el uso de ella es muy fácil.

Para el trabajo colaborativo, Github usa el modo distribuido, lo que quiere decir que el repositorio en el cual quiere trabajar el desarrollador se guarda de manera local y puede ser subido cada cierto tiempo dependiendo de los avances que se tengan. Esto da un alto grado de independencia en el trabajo, y permite que no se tenga que depender de una conexión a Internet para realizar desarrollo sobre el código de la aplicación, mientras tanto se guarda toda la información en archivos locales.

Aparte de permitirnos compartir información de un repositorio, esta herramienta tambien tiene otras funcionalidades para el trabajo en forma colaborativa:

Posee una herramienta para poder hacer revisión del código en conjunto, lo que me permite poder hacer anotaciones sobre los cambios realizados por otros miembros del equipo antes de una subida en producción.

Documentación y mantenimiento de las diferentes versiones de la aplicacion mediante un wiki.

Me permite ver los cambios realizados por ramas, lo que facilita la revisión de los diferentes cambios en el código del software.

La mayoría de los proyectos de software libre tienen su código compartido por GitHub, así que si estas interesado en mejorar o hacer cambios sobre algún software que conozcas en particular, puedes hacerlo desde tu cliente GitHub en tu computador.
